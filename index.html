Semantic Elements in HTML

A Discussion

Semantic elements help a developer communicate more clearly with browsers by providing more specific descriptions of an element. These elements provide a navigational touchstone that improves user experience and accessibility.

Using semantic tags better describes the location and type of content you wish to apply to a page. For example, instead of using a 'div' tag, which does not specify the navigational placement of the content, it is more specific to use a 'header' or 'footer' tag which clearly states where the content lives on the page. Additionally, semantic elements are helpful in communicating to a wider set of web users by providing added clarity to:

  • Users
  • Other developers
  • Computers and search engines
